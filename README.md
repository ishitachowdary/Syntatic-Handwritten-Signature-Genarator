# Project 2: Handwritten Signature Generation and Verification using Vanilla GAN

This project implements an end-to-end pipeline for offline handwritten signature generation and verification using **Vanilla Generative Adversarial Networks (GANs)** and a **Siamese Convolutional Neural Network (CNN)**. Synthetic signatures generated by GANs are used to augment scarce genuine signature data and improve verification robustness on the CEDAR Offline Signature Dataset. :contentReference[oaicite:1]{index=1}

---

## Project Introduction

Offline handwritten signatures are still widely used in banks, universities, e-governance systems, and legal institutions for identity verification. However, practical signature verification systems suffer from:  
- Limited genuine samples per user  
- Scarcity of skilled forgeries  
- High intra-personal variation in handwriting  

Conventional data augmentation techniques fail to capture realistic stroke-level variations. This project explores the use of a **Vanilla GAN** to learn the distribution of handwritten signatures and generate **realistic synthetic samples**, which are then used to augment training data for a **signature verification model**. :contentReference[oaicite:2]{index=2}

---

## Key Features

- **Generic Vanilla GAN** trained on genuine signatures from multiple users  
- **User-specific Vanilla GANs** fine-tuned per signer to capture intra-personal variation  
- **Siamese CNN–based signature verification model**

- **CEDAR dataset preparation scripts** (raw → structured format)  
- **Streamlit UI** for interactive signature generation  
- **FastAPI service** returning a ZIP of generated signatures

## 1) Data Pipeline & Preprocessing

**Dataset:** CEDAR Offline Signature Dataset

### Preprocessing
- Images converted to grayscale
- Resized to **64 × 64**
- Pixel normalization to **[-1, 1]** (for `tanh`)
- Avoids hard binarization & heavy morphology to preserve stroke continuity

### Dataset preparation
```bash
python scripts/prepare_cedar.py
```
**Creates structure**
``` bash
[z (100)]
  └─ Linear(100 → 512×8×8) + BatchNorm + ReLU
      └─ Unflatten → (512, 8, 8)
          └─ ConvTranspose2d(512 → 256) + BN + ReLU
              └─ (256, 16, 16)
          └─ ConvTranspose2d(256 → 128) + BN + ReLU
              └─ (128, 32, 32)
          └─ ConvTranspose2d(128 → 1) + Tanh
              └─ (1, 64, 64)
```
**2) Vanilla GAN Architecture**
Generator

Input: latent vector z ∈ R¹⁰⁰

Pipeline:
``` bash
[z (100)]
  └─ Linear(100 → 512×8×8) + BatchNorm + ReLU
      └─ Unflatten → (512, 8, 8)
          └─ ConvTranspose2d(512 → 256) + BN + ReLU
              └─ (256, 16, 16)
          └─ ConvTranspose2d(256 → 128) + BN + ReLU
              └─ (128, 32, 32)
          └─ ConvTranspose2d(128 → 1) + Tanh
              └─ (1, 64, 64)
```
**Output**

64×64×1 grayscale image

---
**Discriminator**

Input: (1, 64, 64)
```bash
Conv2d(1 → 128) + LeakyReLU
Conv2d(128 → 256) + BN + LeakyReLU
Conv2d(256 → 512) + BN + LeakyReLU
Flatten → Linear → Sigmoid
```
**Output**

Output

Probability P(real)

**Training Details**

lr = 2e-4

betas = (0.5, 0.999)
